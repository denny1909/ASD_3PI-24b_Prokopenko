## 📋 Опис завдання

У файлі задано координати точок. Необхідно:
1. Вивести точки на екран
2. Побудувати всі можливі тупокутні трикутники, що не мають спільних вершин
3. Максимізувати кількість побудованих фігур

## Алгоритми

### 1. Жадібний алгоритм (Greedy)
**Принцип роботи:**
- Генерує всі можливі тупокутні трикутники
- Сортує їх за площею (від більших до менших)
- Жадібно вибирає трикутники, що не мають спільних вершин

**Складність:**
- Часова: O(n³) для генерації + O(m log m) для сортування
- Просторова: O(m), де m - кількість трикутників
- **Гарантія:** не завжди знаходить оптимальне рішення

### 2. Алгоритм з поверненням (Backtracking)
**Принцип роботи:**
- Генерує всі можливі тупокутні трикутники
- Рекурсивно перебирає всі комбінації
- Використовує відсікання гілок для оптимізації
- Знаходить максимальну кількість трикутників

**Складність:**
- Часова: O(2^m) в гіршому випадку, але з відсіканням працює швидше
- Просторова: O(n) для стеку + O(m) для трикутників
- **Гарантія:** знаходить оптимальне рішення


- Додана легенда з кількістю точок та трикутників

- Анімація прогресу під час виконання алгоритмів

## Критерій тупокутності
Трикутник є **тупокутним**, якщо:
```
c² > a² + b²
```
де c - найдовша сторона, a і b - дві інші сторони.
