## 🎯 Виконання вимог завдання

### 2.1 Метод розв'язку ✅
- Математичний підхід: теорема косинусів для визначення тупого кута
- Комбінаторний підхід: генерація всіх можливих трикутників
- Оптимізаційний підхід: максимізація кількості трикутників

### 2.2 Два алгоритми ✅

#### Алгоритм 1: Жадібний (Greedy)
```
Складність часу:   O(n³ log n)
Складність пам'яті: O(n³)

Переваги:
+ Швидкий
+ Підходить для великих даних
+ Простий

Недоліки:
- Не завжди оптимальний
```

#### Алгоритм 2: Backtracking
```
Складність часу:   O(n³ × 2^(n/3))
Складність пам'яті: O(n³)

Переваги:
+ Знаходить оптимум
+ Ефективний pruning

Недоліки:
- Повільний для >50 точок
```

### 2.3 Класи та об'єкти ✅
- 10 класів/інтерфейсів
- Повне ООП
- Інкапсуляція, поліморфізм, наслідування

### 2.4 Структури даних ✅
1. **ArrayList** - динамічні масиви для точок і трикутників
2. **HashSet** - хеш-таблиця для швидкого пошуку використаних точок
3. **Stack** (неявно) - стек рекурсії в backtracking
