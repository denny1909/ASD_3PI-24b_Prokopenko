# 📚 Технічна документація алгоритмів

## Завдання
Знайти максимальну кількість тупокутних трикутників без спільних вершин з заданого набору точок.

---

## 🔍 Математичне обґрунтування

### Визначення тупокутного трикутника

**Теорема косинусів:**
```
c² = a² + b² - 2ab·cos(γ)
```

Для тупого кута γ > 90°, тому cos(γ) < 0, звідси:
```
c² > a² + b²
```

**Критерій:** Трикутник є тупокутним, якщо квадрат однієї сторони більший за суму квадратів двох інших сторін.

### Перевірка в коді:
```java
public boolean isObtuse() {
    double a2 = side1 * side1;
    double b2 = side2 * side2;
    double c2 = side3 * side3;
    
    return (a2 > b2 + c2) || (b2 > a2 + c2) || (c2 > a2 + b2);
}
```

## 🎯 Алгоритм 1: Жадібний підхід

### Опис
Жадібний алгоритм на кожному кроці вибирає локально оптимальне рішення (трикутник з найбільшою площею), сподіваючись отримати глобально хороший результат.

### Псевдокод
```
function GreedyAlgorithm(points):
    // Крок 1: Генерація всіх тупокутних трикутників
    obtuseTriangles = []
    for each combination of 3 points (i, j, k):
        triangle = Triangle(points[i], points[j], points[k])
        if triangle.isValid() and triangle.isObtuse():
            obtuseTriangles.add(triangle)
    
    // Крок 2: Сортування за площею (від більшої до меншої)
    sort(obtuseTriangles, by area, descending)
    
    // Крок 3: Жадібний вибір
    result = []
    usedPoints = set()
    for each triangle in obtuseTriangles:
        if triangle uses no points from usedPoints:
            result.add(triangle)
            usedPoints.add(triangle.p1, triangle.p2, triangle.p3)
    
    return result
```

### Аналіз складності

**Часова складність:**
1. Генерація трикутників: O(n³)
   - Три вкладені цикли для вибору 3 точок з n
   - Кількість комбінацій: C(n,3) = n(n-1)(n-2)/6 ≈ n³/6

2. Сортування: O(m log m)
   - де m - кількість тупокутних трикутників
   - в гіршому випадку m ≈ n³/6

3. Жадібний вибір: O(m)
   - Один прохід по всіх трикутниках

**Загальна складність: O(n³ + m log m) = O(n³ log n)**

**Просторова складність:**
- Зберігання всіх трикутників: O(n³)
- Множина використаних точок: O(n)
- **Загальна: O(n³)**

### Переваги
✅ Простота реалізації  
✅ Швидкість виконання  
✅ Підходить для великих наборів даних  
✅ Детермінований результат

### Недоліки
❌ Не гарантує оптимальне рішення  
❌ Великі витрати пам'яті (O(n³))

### Приклад роботи
```
Точки: A(0,0), B(5,0), C(2,1), D(8,0), E(6,2), F(3,4)

Крок 1: Знайдені тупокутні трикутники:
  T1(A,B,C) - площа: 2.5
  T2(D,E,F) - площа: 8.0
  T3(A,C,F) - площа: 6.0
  T4(B,D,E) - площа: 3.0

Крок 2: Сортування за площею:
  T2(D,E,F) - 8.0 ← вибираємо
  T3(A,C,F) - 6.0   (має спільну вершину F з T2)
  T4(B,D,E) - 3.0   (має спільні вершини D,E з T2)
  T1(A,B,C) - 2.5 ← вибираємо

Результат: [T2, T1]
```

## 🔄 Алгоритм 2: Backtracking (з поверненням)

### Опис
Алгоритм з поверненням рекурсивно досліджує всі можливі комбінації трикутників, використовуючи pruning (відсікання) для відкидання неперспективних гілок.

### Псевдокод
```
function BacktrackingAlgorithm(points):
    obtuseTriangles = generateAllObtuseTriangles(points)
    bestSolution = []
    usedPoints = set()
    
    backtrack([], 0)
    return bestSolution

function backtrack(currentSolution, startIndex):
    // Оновлюємо найкраще рішення
    if size(currentSolution) > size(bestSolution):
        bestSolution = copy(currentSolution)
    
    // Pruning: якщо неможливо побити рекорд
    maxPossibleTriangles = (remainingTriangles) / 3
    if size(currentSolution) + maxPossibleTriangles <= size(bestSolution):
        return  // відсікання гілки
    
    // Рекурсивний перебір
    for i from startIndex to size(obtuseTriangles):
        triangle = obtuseTriangles[i]
        
        if triangle doesn't use any point from usedPoints:
            // Додаємо трикутник
            currentSolution.add(triangle)
            usedPoints.add(triangle.vertices)
            
            // Рекурсивний виклик
            backtrack(currentSolution, i + 1)
            
            // Відкат (backtrack)
            currentSolution.remove(triangle)
            usedPoints.remove(triangle.vertices)
```

### Аналіз складності

**Часова складність:**
1. Генерація трикутників: O(n³)

2. Backtracking: O(2^(n/3))
   - В гіршому випадку потрібно розглянути всі підмножини трикутників
   - Кожен трикутник використовує 3 точки
   - Максимальна кількість трикутників: n/3
   - Кількість підмножин: 2^(n/3)

**Загальна складність: O(n³ × 2^(n/3))**

З pruning на практиці набагато швидше!

**Просторова складність:**
- Зберігання трикутників: O(n³)
- Стек рекурсії: O(n/3)
- **Загальна: O(n³)**

### Оптимізації

1. **Pruning (відсікання)**
   ```java
   int maxPossibleTriangles = (allTriangles - used) / 3;
   if (current + maxPossibleTriangles <= best) return;
   ```

2. **Обмеження глибини**
   ```java
   if (currentStep > MAX_STEPS) return;
   ```

3. **Ранній вихід**
   - Якщо знайдено рішення з n/3 трикутниками (теоретичний максимум)

### Переваги
✅ Знаходить оптимальне рішення  
✅ Ефективне pruning зменшує час  
✅ Менше використання пам'яті під час виконання

### Недоліки
❌ Експоненційна складність  
❌ Повільний для великих наборів (>50 точок)  
❌ Складніша реалізація

### Приклад роботи
```
Точки: A(0,0), B(5,0), C(2,1), D(8,0), E(6,2), F(3,4)

Дерево рекурсії:
                    []
                   /  \
            [T1]       [T2]
           /    \         \
    [T1,T2]   [T1,T3]    [T2,T1]
       X         X          ✓

X - відсікається через спільні вершини
✓ - валідне рішення

Результат: [T2, T1] (оптимальне)
```

## 🔬 Теоретичні обмеження

### Верхня межа
Максимальна кількість трикутників без спільних вершин: **⌊n/3⌋**

Доказ: Кожен трикутник використовує 3 унікальні точки.

### Нижня межа
Задача є NP-складною (схожа на Maximum Independent Set).

Отже:
- Жадібний алгоритм - поліноміальна апроксимація
- Backtracking - експоненційне точне рішення
